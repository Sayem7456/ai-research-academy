<Callout type="info">Transfer learning reuses a pre-trained model on a new task. Instead of training from scratch, you start with knowledge from a large dataset and adapt it to your specific problem. This is how most practical deep learning works today.</Callout>

## What Problem Does It Solve?

Training deep learning models from scratch requires massive datasets and compute. Transfer learning lets you leverage models trained on millions of examples (like ImageNet) for your task with hundreds or thousands of examples.

**Key insight:** Early layers of neural networks learn universal features (edges, textures) that transfer across tasks. Only later layers need task-specific adaptation.

## Types of Transfer Learning

### Feature Extraction
- Freeze pre-trained layers
- Train only the new classifier head
- Fast, works with small datasets

### Fine-Tuning
- Unfreeze some or all pre-trained layers
- Train with small learning rate
- Better performance, requires more data

### Domain Adaptation
- Adapt from source domain to target domain
- Examples: synthetic → real images, English → Chinese

## When to Use What

| Dataset Size | Similarity to Pre-trained | Strategy |
|--------------|---------------------------|----------|
| Small | High | Feature extraction |
| Small | Low | Fine-tune top layers only |
| Large | High | Fine-tune all layers |
| Large | Low | Train from scratch may be better |

## PyTorch Implementation

### Feature Extraction

```python
import torch
import torch.nn as nn
from torchvision import models

# Load pre-trained ResNet18
model = models.resnet18(weights='IMAGENET1K_V1')

# Freeze all layers
for param in model.parameters():
    param.requires_grad = False

# Replace classifier for your task
num_classes = 5
model.fc = nn.Linear(model.fc.in_features, num_classes)

# Only fc layer is trainable
print(f"Trainable parameters: {sum(p.numel() for p in model.parameters() if p.requires_grad):,}")
print(f"Total parameters: {sum(p.numel() for p in model.parameters()):,}")
```

### Fine-Tuning

```python
model = models.resnet18(weights='IMAGENET1K_V1')

# Replace classifier
model.fc = nn.Linear(model.fc.in_features, 5)

# Unfreeze last few layers
for param in model.layer4.parameters():
    param.requires_grad = True

# Use different learning rates for pre-trained vs new layers
optimizer = torch.optim.Adam([
    {'params': model.layer4.parameters(), 'lr': 1e-5},      # Small LR for pre-trained
    {'params': model.fc.parameters(), 'lr': 1e-3}           # Larger LR for new layer
])
```

### Gradual Unfreezing

```python
class GradualUnfreezer:
    def __init__(self, model, layers):
        self.model = model
        self.layers = layers  # List of layer groups to unfreeze in order
        self.current_layer = 0
        
        # Freeze everything initially
        for param in model.parameters():
            param.requires_grad = False
    
    def unfreeze_next(self):
        if self.current_layer < len(self.layers):
            layer = self.layers[self.current_layer]
            for param in layer.parameters():
                param.requires_grad = True
            self.current_layer += 1
            print(f"Unfroze layer group {self.current_layer}/{len(self.layers)}")

# Usage
model = models.resnet18(weights='IMAGENET1K_V1')
model.fc = nn.Linear(model.fc.in_features, 5)

unfreezer = GradualUnfreezer(model, [
    model.fc,           # Start with classifier only
    model.layer4,       # Then last residual block
    model.layer3,       # Then third residual block
])

# Training loop
for epoch in range(30):
    train(model)
    if epoch % 10 == 0:
        unfreezer.unfreeze_next()
```

### Fine-Tuning with Scheduler

```python
model = models.resnet18(weights='IMAGENET1K_V1')
model.fc = nn.Linear(model.fc.in_features, 5)

optimizer = torch.optim.Adam(model.parameters(), lr=1e-4)

# Warm up then cosine decay
warmup_epochs = 5
total_epochs = 50

def lr_lambda(epoch):
    if epoch < warmup_epochs:
        return epoch / warmup_epochs
    else:
        progress = (epoch - warmup_epochs) / (total_epochs - warmup_epochs)
        return 0.5 * (1 + torch.cos(torch.tensor(progress * 3.14159)).item())

scheduler = torch.optim.lr_scheduler.LambdaLR(optimizer, lr_lambda)

for epoch in range(total_epochs):
    train_one_epoch(model, optimizer, scheduler)
    scheduler.step()
```

### Hugging Face Transformers

```python
from transformers import AutoModelForSequenceClassification, AutoTokenizer

# Load pre-trained BERT
model_name = "bert-base-uncased"
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForSequenceClassification.from_pretrained(
    model_name, 
    num_labels=2,
    ignore_mismatched_sizes=True  # Replace classifier head
)

# Freeze base model
for param in model.bert.parameters():
    param.requires_grad = False

# Only classifier is trainable
trainable = sum(p.numel() for p in model.parameters() if p.requires_grad)
print(f"Trainable: {trainable:,}")  # ~60K (classifier only)
```

## Common Pre-trained Models

### Computer Vision
- **ResNet** (18, 34, 50, 101): General purpose
- **EfficientNet**: Best accuracy/efficiency tradeoff
- **ViT**: Vision Transformer, state-of-the-art

### NLP
- **BERT**: Bidirectional, good for classification
- **GPT**: Autoregressive, good for generation
- **RoBERTa**: Optimized BERT, often better

### Audio
- **Wav2Vec 2.0**: Speech recognition
- **Whisper**: Multilingual speech recognition

## Hyperparameters

### Learning Rate
- Feature extraction: $10^{-3}$ to $10^{-4}$
- Fine-tuning: $10^{-5}$ to $10^{-6}$
- Use learning rate finder

### Batch Size
- Smaller for fine-tuning (preserve pre-trained features)
- 16-32 typical

### Epochs
- Feature extraction: 5-10
- Fine-tuning: 3-10
- Early stopping recommended

## When to Use

**Always use when:**
- Small dataset (fewer than 10k examples)
- Task similar to pre-trained model's task
- Limited compute budget

**Consider training from scratch when:**
- Very large dataset (>100k examples)
- Domain very different from pre-trained model
- Specialized architecture needed

## Key Takeaways

1. Transfer learning is the default approach for most practical deep learning
2. Start with feature extraction, then fine-tune if needed
3. Use smaller learning rates for pre-trained layers
4. Gradual unfreezing prevents catastrophic forgetting
5. Pre-trained models are available for vision (torchvision), NLP (Hugging Face), and audio
6. Always check if a pre-trained model exists for your task before training from scratch
