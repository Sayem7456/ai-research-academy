<Callout type="info">CNNs are the backbone of computer vision. They use learnable filters to detect local patterns (edges, textures, objects) and build hierarchical representations from pixels to concepts.</Callout>

## What Problem Does It Solve?

Processing images with fully connected layers is impractical — a 224x224x3 image has 150,528 features per pixel. CNNs exploit spatial structure through weight sharing and local connectivity, drastically reducing parameters.

**Key insight:** The same filter detects the same pattern everywhere in the image (translation invariance), and stacking filters builds increasingly complex features (hierarchy).

## Mathematical Intuition

### Convolution Operation

$$
(f * g)(i, j) = \sum_{m}\sum_{n} f(m, n) \cdot g(i-m, j-n)
$$

In CNNs, a small filter slides over the input, computing dot products at each position.

### Output Size

$$
\text{Output size} = \frac{\text{Input size} - \text{Filter size} + 2 \times \text{Padding}}{\text{Stride}} + 1
$$

### Pooling

Reduces spatial dimensions:
- **Max pooling:** Takes maximum value in each window
- **Average pooling:** Takes mean value in each window

## CNN Architecture

```
Input Image → [Conv → ReLU → Pool] × N → Flatten → [FC → ReLU] × M → Output
```

### Convolutional Layer
- Detects local features (edges, corners, textures)
- Multiple filters = multiple feature maps
- Each filter learns different patterns

### Pooling Layer
- Reduces spatial dimensions
- Provides translation invariance
- Reduces computation

### Fully Connected Layer
- Combines features for classification
- Usually at the end of the network

## PyTorch Implementation

### Basic CNN

```python
import torch
import torch.nn as nn

class SimpleCNN(nn.Module):
    def __init__(self, num_classes=10):
        super().__init__()
        self.features = nn.Sequential(
            # Conv Block 1: 3 -> 32 channels
            nn.Conv2d(3, 32, kernel_size=3, padding=1),
            nn.BatchNorm2d(32),
            nn.ReLU(),
            nn.MaxPool2d(2, 2),                    # 32x32 -> 16x16
            
            # Conv Block 2: 32 -> 64 channels
            nn.Conv2d(32, 64, kernel_size=3, padding=1),
            nn.BatchNorm2d(64),
            nn.ReLU(),
            nn.MaxPool2d(2, 2),                    # 16x16 -> 8x8
            
            # Conv Block 3: 64 -> 128 channels
            nn.Conv2d(64, 128, kernel_size=3, padding=1),
            nn.BatchNorm2d(128),
            nn.ReLU(),
            nn.AdaptiveAvgPool2d((4, 4))           # Any -> 4x4
        )
        
        self.classifier = nn.Sequential(
            nn.Flatten(),
            nn.Linear(128 * 4 * 4, 256),
            nn.ReLU(),
            nn.Dropout(0.5),
            nn.Linear(256, num_classes)
        )
    
    def forward(self, x):
        x = self.features(x)
        x = self.classifier(x)
        return x

model = SimpleCNN(num_classes=10)
x = torch.randn(1, 3, 32, 32)  # batch=1, channels=3, H=32, W=32
print(model(x).shape)           # torch.Size([1, 10])
```

### Conv2d Parameters

```python
# Key parameters
conv = nn.Conv2d(
    in_channels=3,       # Input channels (RGB=3)
    out_channels=64,     # Number of filters
    kernel_size=3,       # Filter size (3x3)
    stride=1,            # Step size
    padding=1,           # Zero-padding (same padding)
    bias=True            # Add bias term
)

# Calculate parameters: (3 * 3 * 3 + 1) * 64 = 1,792
print(f"Parameters: {sum(p.numel() for p in conv.parameters()):,}")
```

### Feature Map Visualization

```python
# Extract intermediate feature maps
class FeatureExtractor(nn.Module):
    def __init__(self, model, layers):
        super().__init__()
        self.model = model
        self.layers = layers
        self.features = {}
        
        for name, layer in self.model.named_modules():
            if name in layers:
                layer.register_forward_hook(self._hook(name))
    
    def _hook(self, name):
        def hook(module, input, output):
            self.features[name] = output
        return hook
    
    def forward(self, x):
        self.model(x)
        return self.features

# Usage
extractor = FeatureExtractor(model, ['features.0', 'features.3', 'features.6'])
features = extractor(x)
```

## Common Architectures

### LeNet-5 (1998)
- First successful CNN
- 2 conv layers, 3 FC layers
- Used for digit recognition

### AlexNet (2012)
- Won ImageNet competition
- 5 conv layers, 3 FC layers
- Introduced ReLU and Dropout

### VGGNet (2014)
- Very deep (16-19 layers)
- Small 3x3 filters throughout
- Simple, uniform architecture

### ResNet (2015)
- Skip connections enable training of 100+ layer networks
- Residual blocks: $y = F(x) + x$
- Solved vanishing gradient problem

## Hyperparameters

### Kernel Size
- 3x3: most common, good balance
- 1x1: pointwise convolution, channel mixing
- 5x5: larger receptive field, more parameters

### Stride
- 1: standard convolution
- 2: downsampling (replaces pooling)

### Padding
- "same": output same size as input
- "valid": no padding

## When to Use

**Good for:**
- Image classification
- Object detection
- Semantic segmentation
- Any grid-structured data (time series, spectrograms)

**Avoid for:**
- Variable-length sequences (use RNNs/Transformers)
- Very small datasets (use transfer learning)

## Key Takeaways

1. CNNs use local connectivity and weight sharing to process images efficiently
2. Convolution detects local patterns; pooling provides translation invariance
3. Stack conv blocks to build hierarchical features (edges → textures → parts → objects)
4. 3x3 filters with padding=1 and stride=1 are the standard building block
5. ResNet skip connections enable training very deep networks
6. Use `nn.AdaptiveAvgPool2d` to handle variable input sizes
