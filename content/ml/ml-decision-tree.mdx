<Callout type="info">Decision Trees learn hierarchical decision rules from data, creating an intuitive tree structure that mirrors human decision-making processes.</Callout>

## What Problem Does It Solve?

Decision Trees solve **classification** and **regression** problems by learning a series of if-then-else rules that partition the feature space into regions. They handle non-linear relationships naturally without requiring feature engineering.

**Real-world applications:**
- Medical diagnosis (symptom-based decision pathways)
- Credit scoring (loan approval criteria)
- Customer segmentation (marketing strategy decisions)
- Fraud detection (transaction rule patterns)

## Mathematical Intuition

### The Tree Structure

A decision tree consists of:
- **Root node**: Contains all training data
- **Internal nodes**: Decision rules (e.g., "Age `< 30`?")
- **Branches**: Outcomes of decisions (Yes/No)
- **Leaf nodes**: Final predictions (class labels or values)

### How Trees Split Data

At each node, the algorithm chooses the feature and threshold that best splits the data. "Best" is measured by **impurity reduction**.

### Classification: Gini Impurity

Measures how often a randomly chosen element would be incorrectly labeled:

$$
\text{Gini}(S) = 1 - \sum_{i=1}^{C} p_i^2
$$

Where $p_i$ is the proportion of class $i$ in set $S$.

**Examples:**
- Pure node (all one class): Gini = 0
- Binary 50/50 split: Gini = 0.5

**Split quality:**
$$
\text{Gain}_{\text{Gini}} = \text{Gini}(S) - \sum_{i=1}^{k} \frac{|S_i|}{|S|} \text{Gini}(S_i)
$$

### Classification: Entropy (Information Gain)

Measures disorder or uncertainty:

$$
\text{Entropy}(S) = -\sum_{i=1}^{C} p_i \log_2(p_i)
$$

**Information Gain:**
$$
\text{IG}(S, A) = \text{Entropy}(S) - \sum_{v \in \text{Values}(A)} \frac{|S_v|}{|S|} \text{Entropy}(S_v)
$$

Choose the feature that maximizes information gain.

### Regression: Variance Reduction

For regression trees, measure split quality by variance reduction:

$$
\text{Var}(S) = \frac{1}{|S|} \sum_{i \in S} (y_i - \bar{y})^2
$$

$$
\text{Gain}_{\text{Var}} = \text{Var}(S) - \sum_{i=1}^{k} \frac{|S_i|}{|S|} \text{Var}(S_i)
$$

Leaf prediction is the mean of samples: $\hat{y} = \frac{1}{|S|} \sum_{i \in S} y_i$

### Recursive Splitting Algorithm

```
function BuildTree(data, features):
    if stopping_criterion_met(data):
        return LeafNode(majority_class or mean_value)
    
    best_feature, best_threshold = find_best_split(data, features)
    left_data = data[feature <= threshold]
    right_data = data[feature > threshold]
    
    left_subtree = BuildTree(left_data, features)
    right_subtree = BuildTree(right_data, features)
    
    return DecisionNode(best_feature, best_threshold, left_subtree, right_subtree)
```

## Implementation

### From Scratch (NumPy)

```python
import numpy as np
from collections import Counter

class Node:
    def __init__(self, feature=None, threshold=None, left=None, right=None, value=None):
        self.feature = feature
        self.threshold = threshold
        self.left = left
        self.right = right
        self.value = value
    
    def is_leaf(self):
        return self.value is not None

class DecisionTree:
    def __init__(self, max_depth=10, min_samples_split=2, criterion='gini'):
        self.max_depth = max_depth
        self.min_samples_split = min_samples_split
        self.criterion = criterion
        self.root = None
    
    def _gini(self, y):
        m = len(y)
        return 1.0 - sum((np.sum(y == c) / m) ** 2 for c in np.unique(y))
    
    def _entropy(self, y):
        m = len(y)
        entropy = 0
        for c in np.unique(y):
            p = np.sum(y == c) / m
            entropy -= p * np.log2(p + 1e-9)
        return entropy
    
    def _variance(self, y):
        return np.var(y) if len(y) > 0 else 0
    
    def _split(self, X, y, feature, threshold):
        left_mask = X[:, feature] <= threshold
        right_mask = ~left_mask
        return X[left_mask], X[right_mask], y[left_mask], y[right_mask]
    
    def _best_split(self, X, y):
        best_gain = -1
        best_feature = None
        best_threshold = None
        
        n_samples, n_features = X.shape
        parent_impurity = self._gini(y) if self.criterion == 'gini' else self._entropy(y)
        
        for feature in range(n_features):
            thresholds = np.unique(X[:, feature])
            
            for threshold in thresholds:
                X_left, X_right, y_left, y_right = self._split(X, y, feature, threshold)
                
                if len(y_left) == 0 or len(y_right) == 0:
                    continue
                
                if self.criterion == 'gini':
                    left_impurity = self._gini(y_left)
                    right_impurity = self._gini(y_right)
                else:
                    left_impurity = self._entropy(y_left)
                    right_impurity = self._entropy(y_right)
                
                n_left, n_right = len(y_left), len(y_right)
                child_impurity = (n_left / n_samples) * left_impurity + (n_right / n_samples) * right_impurity
                gain = parent_impurity - child_impurity
                
                if gain > best_gain:
                    best_gain = gain
                    best_feature = feature
                    best_threshold = threshold
        
        return best_feature, best_threshold
    
    def _build_tree(self, X, y, depth=0):
        n_samples, n_features = X.shape
        n_classes = len(np.unique(y))
        
        if depth >= self.max_depth or n_samples < self.min_samples_split or n_classes == 1:
            leaf_value = Counter(y).most_common(1)[0][0]
            return Node(value=leaf_value)
        
        feature, threshold = self._best_split(X, y)
        
        if feature is None:
            leaf_value = Counter(y).most_common(1)[0][0]
            return Node(value=leaf_value)
        
        X_left, X_right, y_left, y_right = self._split(X, y, feature, threshold)
        
        left_subtree = self._build_tree(X_left, y_left, depth + 1)
        right_subtree = self._build_tree(X_right, y_right, depth + 1)
        
        return Node(feature=feature, threshold=threshold, left=left_subtree, right=right_subtree)
    
    def fit(self, X, y):
        self.root = self._build_tree(X, y)
    
    def _traverse_tree(self, x, node):
        if node.is_leaf():
            return node.value
        
        if x[node.feature] <= node.threshold:
            return self._traverse_tree(x, node.left)
        else:
            return self._traverse_tree(x, node.right)
    
    def predict(self, X):
        return np.array([self._traverse_tree(x, self.root) for x in X])
    
    def score(self, X, y):
        predictions = self.predict(X)
        return np.mean(predictions == y)

from sklearn.datasets import make_classification
from sklearn.model_selection import train_test_split

X, y = make_classification(n_samples=500, n_features=10, random_state=42)
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

tree = DecisionTree(max_depth=5, min_samples_split=10, criterion='gini')
tree.fit(X_train, y_train)

print(f"Train Accuracy: {tree.score(X_train, y_train):.4f}")
print(f"Test Accuracy: {tree.score(X_test, y_test):.4f}")
```

from sklearn.tree import DecisionTreeClassifier, plot_tree

## Hyperparameters

### max_depth
Controls tree depth. Deeper trees = more complex = higher overfitting risk.

### min_samples_split
Minimum samples required to split a node. Higher values prevent overfitting.

### min_samples_leaf
Minimum samples required in a leaf. Ensures leaves represent meaningful patterns.

### max_features
Number of features to consider for each split. Adds randomness, reduces overfitting.

## Feature Importance

Decision trees naturally provide feature importance:

$$
\text{Importance}(f) = \sum_{\text{nodes using } f} \frac{n_{\text{samples}}}{n_{\text{total}}} \times \text{impurity reduction}
$$

```python
importances = tree.feature_importances_
for i, imp in enumerate(importances):
    print(f"Feature {i}: {imp:.4f}")
```

## Limitations

<Callout type="warning">Decision trees are prone to overfitting and instability. Small changes in data can produce completely different trees.</Callout>

### 1. High Variance (Overfitting)
Without constraints, trees memorize training data perfectly (100% training accuracy, poor test accuracy).

### 2. Instability
Small data changes can drastically alter tree structure. Not robust.

### 3. Biased to Dominant Classes
Imbalanced datasets lead to biased trees. Solution: class weights.

### 4. Greedy Algorithm
Makes locally optimal splits, not globally optimal tree. Cannot backtrack.

### 5. Biased to High-Cardinality Features
Features with many unique values (e.g., IDs) get unfairly high importance.

### 6. Poor Extrapolation
Predictions outside training range default to nearest leaf value.

## Advantages

### 1. Interpretability
Easy to visualize and explain to non-technical stakeholders.

### 2. No Feature Scaling Required
Trees don't care about feature magnitudes.

### 3. Handles Non-Linearity
Captures complex interactions without manual feature engineering.

### 4. Mixed Data Types
Handles numerical and categorical features naturally.

### 5. Feature Importance
Automatically identifies relevant features.

## Modern Improvements

### Ensemble Methods

**Random Forest**: Average many randomized trees
**Gradient Boosting**: Sequentially build trees to correct errors (XGBoost, LightGBM, CatBoost)
**AdaBoost**: Weight training samples based on previous errors

### Pruning

**Pre-pruning**: Stop growing early (max_depth, min_samples_split)
**Post-pruning**: Grow full tree, then remove branches that don't improve validation performance

## Research Notes

### Historical Context

**1984** - Breiman, Friedman, Olshen, Stone published "Classification and Regression Trees" (CART)
**1986** - Quinlan developed ID3 algorithm (Information Gain)
**1993** - Quinlan improved to C4.5 algorithm

### Theoretical Properties

**CART Theorem**: Finds the split that maximizes impurity reduction at each step (greedy optimization).

**Bias-Variance**: High variance, low bias. Prone to overfitting but can capture complex patterns.

### Modern Applications

Single trees are rarely used in production. Instead:
- **Baseline model**: Quick interpretable model before trying ensembles
- **Feature selection**: Use importance scores to guide feature engineering
- **Interpretability**: Train tree on complex model's predictions to explain black-box models (model distillation)

### Why Ensembles Dominate

- **Random Forest**: Reduces variance through averaging
- **Gradient Boosting**: Reduces bias through sequential error correction
- **Both**: Achieve state-of-the-art results on tabular data

<Callout type="tip">
Use single decision trees for interpretability and exploration. For production models, use Random Forest or Gradient Boosting variants.
</Callout>
