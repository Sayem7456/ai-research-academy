<Callout type="info">Random Forest is an ensemble algorithm that combines multiple decision trees with randomness to create a robust, high-performance model that reduces overfitting.</Callout>

## What Problem Does It Solve?

Random Forest solves the **instability and overfitting** problems of single decision trees by building many trees on random subsets of data and features, then aggregating their predictions.

**Real-world applications:**
- Credit scoring (combines multiple risk factors)
- Medical diagnosis (aggregates multiple symptom patterns)
- Fraud detection (ensemble of transaction rules)
- Feature ranking (consensus importance across trees)

## Mathematical Intuition

### Ensemble Learning Principle

Instead of relying on one model, combine many:

$$
\hat{y} = \frac{1}{T} \sum_{t=1}^{T} h_t(\mathbf{x})
$$

Where $T$ is the number of trees and $h_t$ is the prediction of tree $t$.

### Two Sources of Randomness

**1. Bootstrap Aggregating (Bagging)**

Sample with replacement to create diverse training sets:
- Original dataset: $n$ samples
- Each tree trains on: $n$ samples (with duplicates)
- ~63.2% unique samples per tree
- Remaining ~36.8% = "out-of-bag" (OOB) samples for validation

**2. Random Feature Selection**

At each split, consider only $m$ random features instead of all $p$ features:
- Classification: $m = \sqrt{p}$
- Regression: $m = p/3$
- Decorrelates trees, reduces variance

### Algorithm

```
function RandomForest(data, n_trees, max_features):
    forest = []
    
    for i in 1 to n_trees:
        bootstrap_sample = sample_with_replacement(data)
        tree = DecisionTree(max_features=max_features)
        tree.fit(bootstrap_sample)
        forest.append(tree)
    
    return forest

function Predict(forest, x):
    predictions = [tree.predict(x) for tree in forest]
    
    if classification:
        return majority_vote(predictions)
    else:
        return mean(predictions)
```

### Variance Reduction

For independent trees with variance $\sigma^2$:

$$
\text{Var}(\text{average}) = \frac{\sigma^2}{T}
$$

As $T \to \infty$, variance $\to 0$.

In practice, trees are correlated, so variance reduction is less but still significant.

## Implementation

### From Scratch (NumPy)

```python
import numpy as np
from collections import Counter

class RandomForestClassifier:
    def __init__(self, n_trees=100, max_depth=10, min_samples_split=2, max_features='sqrt'):
        self.n_trees = n_trees
        self.max_depth = max_depth
        self.min_samples_split = min_samples_split
        self.max_features = max_features
        self.trees = []
    
    def _bootstrap_sample(self, X, y):
        n_samples = X.shape[0]
        indices = np.random.choice(n_samples, n_samples, replace=True)
        return X[indices], y[indices]
    
    def _get_max_features(self, n_features):
        if self.max_features == 'sqrt':
            return int(np.sqrt(n_features))
        elif self.max_features == 'log2':
            return int(np.log2(n_features))
        elif isinstance(self.max_features, int):
            return self.max_features
        else:
            return n_features
    
    def fit(self, X, y):
        self.trees = []
        n_features = X.shape[1]
        max_features = self._get_max_features(n_features)
        
        for _ in range(self.n_trees):
            tree = DecisionTree(
                max_depth=self.max_depth,
                min_samples_split=self.min_samples_split,
                max_features=max_features
            )
            X_sample, y_sample = self._bootstrap_sample(X, y)
            tree.fit(X_sample, y_sample)
            self.trees.append(tree)
    
    def predict(self, X):
        tree_predictions = np.array([tree.predict(X) for tree in self.trees])
        
        predictions = []
        for i in range(X.shape[0]):
            pred = Counter(tree_predictions[:, i]).most_common(1)[0][0]
            predictions.append(pred)
        
        return np.array(predictions)
    
    def predict_proba(self, X):
        tree_predictions = np.array([tree.predict(X) for tree in self.trees])
        n_samples = X.shape[0]
        
        probas = []
        for i in range(n_samples):
            class_counts = Counter(tree_predictions[:, i])
            proba = {cls: count / self.n_trees for cls, count in class_counts.items()}
            probas.append(proba)
        
        return probas
    
    def score(self, X, y):
        predictions = self.predict(X)
        return np.mean(predictions == y)
```

### Using Scikit-Learn

```python
from sklearn.ensemble import RandomForestClassifier, RandomForestRegressor
from sklearn.datasets import make_classification
from sklearn.model_selection import train_test_split, cross_val_score
from sklearn.metrics import classification_report, confusion_matrix

X, y = make_classification(n_samples=1000, n_features=20, random_state=42)
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

rf = RandomForestClassifier(
    n_estimators=100,
    max_depth=10,
    min_samples_split=5,
    max_features='sqrt',
    random_state=42,
    n_jobs=-1
)

rf.fit(X_train, y_train)

print(f"Train Accuracy: {rf.score(X_train, y_train):.4f}")
print(f"Test Accuracy: {rf.score(X_test, y_test):.4f}")

cv_scores = cross_val_score(rf, X_train, y_train, cv=5)
print(f"CV Accuracy: {cv_scores.mean():.4f} (+/- {cv_scores.std():.4f})")

print(f"\nTop 5 Feature Importances:")
feature_importance = sorted(zip(range(20), rf.feature_importances_), key=lambda x: x[1], reverse=True)
for idx, importance in feature_importance[:5]:
    print(f"  Feature {idx}: {importance:.4f}")
```

## Out-of-Bag (OOB) Error

Each tree is trained on ~63% of data. The remaining ~37% can be used for validation without a separate test set:

$$
\text{OOB Error} = \frac{1}{n} \sum_{i=1}^{n} \mathbb{1}[\text{OOB prediction}_i \neq y_i]
$$

```python
rf = RandomForestClassifier(n_estimators=100, oob_score=True, random_state=42)
rf.fit(X_train, y_train)
print(f"OOB Score: {rf.oob_score_:.4f}")
```

OOB score approximates cross-validation without extra computation.

## Feature Importance

Random Forest provides robust feature importance by averaging across trees:

$$
\text{Importance}(f) = \frac{1}{T} \sum_{t=1}^{T} \text{Importance}_t(f)
$$

```python
import matplotlib.pyplot as plt

importances = rf.feature_importances_
indices = np.argsort(importances)[::-1]

plt.figure(figsize=(10, 6))
plt.title("Feature Importances")
plt.bar(range(X.shape[1]), importances[indices])
plt.xlabel("Feature Index")
plt.ylabel("Importance")
plt.show()
```

## Hyperparameters

### n_estimators (number of trees)
- More trees = better performance (diminishing returns after ~100-500)
- No overfitting risk (more is better)
- Tradeoff: training time and memory

### max_depth
- Controls tree complexity
- Deeper = more expressive but slower

### min_samples_split / min_samples_leaf
- Regularization: prevents tiny splits
- Higher = simpler trees

### max_features
- Controls randomness per split
- 'sqrt': good for classification
- 'log2': alternative for classification
- 1/3: good for regression

### max_samples (Bootstrap sample size)
- Default: 1.0 (100% of data, with replacement)
- Lower values increase diversity but may underfit

## Advantages

### 1. Robust to Overfitting
Averaging many trees reduces variance dramatically.

### 2. Handles High-Dimensional Data
Works well even when features >> samples.

### 3. Feature Importance
Automatically ranks feature relevance.

### 4. No Feature Scaling Required
Trees don't care about feature magnitudes.

### 5. Handles Missing Values
Scikit-learn's implementation handles NaNs gracefully.

### 6. Parallel Training
Trees are independent, train in parallel (fast).

### 7. OOB Evaluation
Built-in validation without test set.

## Limitations

<Callout type="warning">Random Forest trades interpretability for performance. Individual tree decisions are lost in the ensemble.</Callout>

### 1. Black Box Model
Cannot easily explain individual predictions (unlike single trees).

### 2. Memory and Speed
Stores many trees, slower prediction than single models.

### 3. Poor Extrapolation
Like decision trees, cannot predict outside training range.

### 4. Biased to Imbalanced Data
Needs class weights or balanced sampling.

### 5. Not Ideal for Linear Relationships
Overkill for simple linear patterns.

## Comparison: Random Forest vs Single Tree

| Aspect | Single Tree | Random Forest |
|--------|-------------|---------------|
| Variance | High | Low |
| Bias | Low | Low |
| Overfitting | Prone | Resistant |
| Interpretability | High | Low |
| Speed | Fast | Slower |
| Accuracy | Lower | Higher |

## Modern Improvements

### Extremely Randomized Trees (Extra Trees)
Splits are fully random (not optimized), even faster training.

### Gradient Boosting
Sequential trees that correct errors: XGBoost, LightGBM, CatBoost.

### Isolation Forest
Random Forest variant for anomaly detection.

## Research Notes

### Historical Context

**2001** - Leo Breiman published "Random Forests" in Machine Learning journal

Built on earlier work:
- Bagging (Breiman, 1996)
- Random Subspace Method (Ho, 1998)

### Theoretical Guarantees

**Breiman's Law of Large Numbers for Random Forests**: As $T \to \infty$, generalization error converges to:

$$
PE^* = \bar{\rho} \sigma^2 (1 - s^2)
$$

Where:
- $\bar{\rho}$ = average correlation between trees
- $\sigma^2$ = variance of individual trees
- $s$ = strength of individual trees

**Key insight**: Lower correlation + stronger trees = better forest.

### Why It Dominates Tabular Data

Before deep learning, Random Forest was the default choice for structured data. Even today:
- **Kaggle competitions**: Often wins on tabular datasets
- **Interpretability**: Feature importance aids understanding
- **Robustness**: Works out-of-the-box with minimal tuning

### When to Use Random Forest

**Good for:**
- Tabular data with mixed feature types
- Non-linear relationships
- High-dimensional data
- When you need feature importance
- Quick baseline model

**Avoid for:**
- Very large datasets (consider LightGBM)
- Linear relationships (use linear models)
- Image/text data (use deep learning)
- When interpretability is critical (use single tree or linear model)

<Callout type="tip">
Random Forest is often the first model to try on a new tabular dataset. If it performs well, you have a strong baseline. If it fails, the data may be unsuitable for tree-based methods.
</Callout>
