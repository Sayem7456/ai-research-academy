<Callout type="info">K-Means is an unsupervised clustering algorithm that partitions data into K distinct groups by iteratively assigning points to the nearest centroid.</Callout>

## What Problem Does It Solve?

K-Means solves **clustering problems** by grouping similar data points together without labeled data. It discovers natural structure in data by minimizing within-cluster variance.

**Real-world applications:**
- Customer segmentation (marketing groups)
- Image compression (color quantization)
- Document clustering (topic grouping)
- Anomaly detection (points far from clusters)

## Mathematical Intuition

### The Algorithm

Given data $\mathbf{X} = \{\mathbf{x}_1, ..., \mathbf{x}_n\}$ and desired $K$ clusters:

**Initialize**: Randomly select $K$ centroids $\boldsymbol{\mu}_1, ..., \boldsymbol{\mu}_K$

**Repeat until convergence:**

1. **Assignment Step**: Assign each point to nearest centroid
   $$
   c_i = \arg\min_{k} \|\mathbf{x}_i - \boldsymbol{\mu}_k\|^2
   $$

2. **Update Step**: Recompute centroids as cluster means
   $$
   \boldsymbol{\mu}_k = \frac{1}{|C_k|} \sum_{\mathbf{x}_i \in C_k} \mathbf{x}_i
   $$

Where $C_k$ is the set of points assigned to cluster $k$.

### Objective Function

K-Means minimizes **within-cluster sum of squares (WCSS)**:

$$
J = \sum_{k=1}^{K} \sum_{\mathbf{x}_i \in C_k} \|\mathbf{x}_i - \boldsymbol{\mu}_k\|^2
$$

**Convergence**: Algorithm stops when centroids don't change or max iterations reached.

### Distance Metric

Standard K-Means uses **Euclidean distance**:

$$
d(\mathbf{x}_i, \boldsymbol{\mu}_k) = \sqrt{\sum_{j=1}^{d} (x_{ij} - \mu_{kj})^2}
$$

Alternatives: Manhattan, Cosine similarity (for text data).

## Implementation

### From Scratch (NumPy)

```python
import numpy as np

class KMeans:
    def __init__(self, n_clusters=3, max_iters=100, random_state=None):
        self.n_clusters = n_clusters
        self.max_iters = max_iters
        self.random_state = random_state
        self.centroids = None
        self.labels = None
        
    def fit(self, X):
        np.random.seed(self.random_state)
        n_samples, n_features = X.shape
        
        random_indices = np.random.choice(n_samples, self.n_clusters, replace=False)
        self.centroids = X[random_indices]
        
        for iteration in range(self.max_iters):
            old_centroids = self.centroids.copy()
            
            distances = np.zeros((n_samples, self.n_clusters))
            for k in range(self.n_clusters):
                distances[:, k] = np.linalg.norm(X - self.centroids[k], axis=1)
            
            self.labels = np.argmin(distances, axis=1)
            
            for k in range(self.n_clusters):
                cluster_points = X[self.labels == k]
                if len(cluster_points) > 0:
                    self.centroids[k] = cluster_points.mean(axis=0)
            
            if np.allclose(old_centroids, self.centroids):
                print(f"Converged at iteration {iteration}")
                break
        
        return self
    
    def predict(self, X):
        distances = np.zeros((len(X), self.n_clusters))
        for k in range(self.n_clusters):
            distances[:, k] = np.linalg.norm(X - self.centroids[k], axis=1)
        return np.argmin(distances, axis=1)
    
    def inertia(self, X):
        distances = np.linalg.norm(X - self.centroids[self.labels], axis=1)
        return np.sum(distances ** 2)

from sklearn.datasets import make_blobs

X, y_true = make_blobs(n_samples=300, centers=4, random_state=42)

kmeans = KMeans(n_clusters=4, random_state=42)
kmeans.fit(X)

print(f"Inertia: {kmeans.inertia(X):.2f}")
print(f"Centroids:\n{kmeans.centroids}")
```

### Using Scikit-Learn

```python
from sklearn.cluster import KMeans
from sklearn.datasets import make_blobs
from sklearn.preprocessing import StandardScaler
import matplotlib.pyplot as plt

X, _ = make_blobs(n_samples=500, centers=3, random_state=42)

scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

kmeans = KMeans(n_clusters=3, init='k-means++', n_init=10, max_iter=300, random_state=42)
kmeans.fit(X_scaled)

labels = kmeans.labels_
centroids = kmeans.cluster_centers_

print(f"Inertia: {kmeans.inertia_:.2f}")
print(f"Iterations: {kmeans.n_iter_}")

plt.figure(figsize=(10, 6))
plt.scatter(X_scaled[:, 0], X_scaled[:, 1], c=labels, cmap='viridis', alpha=0.6)
plt.scatter(centroids[:, 0], centroids[:, 1], c='red', marker='X', s=200, edgecolors='black')
plt.title('K-Means Clustering')
plt.show()
```

## Choosing K: Elbow Method

Plot inertia vs number of clusters. Look for "elbow" where decrease slows.

```python
inertias = []
K_range = range(1, 11)

for k in K_range:
    kmeans = KMeans(n_clusters=k, random_state=42)
    kmeans.fit(X_scaled)
    inertias.append(kmeans.inertia_)

plt.figure(figsize=(10, 6))
plt.plot(K_range, inertias, 'bo-')
plt.xlabel('Number of Clusters (K)')
plt.ylabel('Inertia')
plt.title('Elbow Method')
plt.grid()
plt.show()
```

### Silhouette Score

Measures how similar points are to their own cluster vs other clusters:

$$
s_i = \frac{b_i - a_i}{\max(a_i, b_i)}
$$

Where $a_i$ = average distance to points in same cluster, $b_i$ = average distance to nearest other cluster.

Score ranges from -1 (wrong cluster) to +1 (perfect clustering).

```python
from sklearn.metrics import silhouette_score

for k in range(2, 11):
    kmeans = KMeans(n_clusters=k, random_state=42)
    labels = kmeans.fit_predict(X_scaled)
    score = silhouette_score(X_scaled, labels)
    print(f"K={k}: Silhouette Score = {score:.4f}")
```

## Initialization Methods

### Random Initialization
- Fast but unstable
- Different runs produce different results
- May converge to poor local minima

### K-Means++ (Default in Scikit-Learn)
- Smart initialization: spread out initial centroids
- Better convergence, more stable results
- Slightly slower initialization

Algorithm:
1. Choose first centroid randomly
2. For each subsequent centroid, choose point with probability proportional to distance from nearest existing centroid
3. Repeat until K centroids selected

## Complexity Analysis

### Time Complexity
- Per iteration: $O(nKd)$ where $n$ = samples, $K$ = clusters, $d$ = dimensions
- Total: $O(IKnd)$ where $I$ = iterations (typically $< 100$)

### Space Complexity
- $O(nd + Kd)$ for data and centroids

**Scalable**: Works well with large datasets (millions of points).

## Advantages

### 1. Simple and Intuitive
Easy to understand and implement.

### 2. Fast and Scalable
Efficient for large datasets.

### 3. Guaranteed Convergence
Always converges (but possibly to local minimum).

### 4. Works Well with Spherical Clusters
Effective when clusters are roughly equal-sized and spherical.

## Limitations

<Callout type="warning">K-Means assumes spherical clusters of similar size and requires specifying K in advance.</Callout>

### 1. Must Specify K
Requires knowing number of clusters beforehand.

### 2. Sensitive to Initialization
Different initializations give different results (use K-Means++).

### 3. Assumes Spherical Clusters
Fails on elongated or irregular shapes.

### 4. Sensitive to Outliers
Outliers distort centroid positions.

### 5. Assumes Equal-Sized Clusters
Struggles with imbalanced cluster sizes.

### 6. Euclidean Distance Assumption
Requires feature scaling; doesn't work well for categorical data.

## When to Use K-Means

**Good for:**
- Customer segmentation (marketing)
- Image compression (color quantization)
- Data preprocessing (cluster-based features)
- Quick exploratory clustering
- Large datasets (fast and scalable)

**Avoid for:**
- Non-spherical cluster shapes (use DBSCAN, Gaussian Mixture)
- Varying cluster densities (use HDBSCAN)
- Unknown number of clusters (use hierarchical clustering)
- Categorical data (use K-Modes)

## Modern Alternatives

### Mini-Batch K-Means
Uses random samples per iteration. Much faster for huge datasets.

### DBSCAN
Density-based clustering, no need to specify K, handles arbitrary shapes.

### Gaussian Mixture Models
Probabilistic soft clustering with EM algorithm.

### Hierarchical Clustering
Creates dendrogram, no need to specify K upfront.

## Research Notes

### Historical Context

**1957** - Stuart Lloyd developed K-Means at Bell Labs (published 1982)
**1965** - MacQueen coined the term "K-Means"

One of the oldest and most widely used clustering algorithms.

### Theoretical Properties

**NP-Hard**: Finding global optimum is NP-hard, but algorithm finds good local minima quickly.

**Voronoi Partitioning**: K-Means creates Voronoi diagram where each region corresponds to one cluster.

### Modern Applications

**Computer Vision**: Image segmentation, color quantization
**Recommendation Systems**: User grouping for collaborative filtering
**Data Compression**: Vector quantization
**Preprocessing**: Feature engineering (distance to cluster centers)

<Callout type="tip">
K-Means is the first clustering algorithm to try. It's fast, simple, and often works well. If it fails, consider DBSCAN for non-spherical clusters or hierarchical clustering when K is unknown.
</Callout>
