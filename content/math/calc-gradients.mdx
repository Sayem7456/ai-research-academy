<Callout type="info">The gradient vector points in the direction of steepest ascent — the cornerstone of optimization in machine learning. Gradient descent follows the negative gradient to find minima.</Callout>

## What Problem Does It Solve?

A loss function depends on millions of parameters. Which direction should we move to decrease the loss fastest? The gradient tells us exactly that.

## Definition

The gradient of $f(x_1, x_2, \ldots, x_n)$ is the vector of all partial derivatives:

$$
\nabla f = \begin{bmatrix} \frac{\partial f}{\partial x_1} \\ \frac{\partial f}{\partial x_2} \\ \vdots \\ \frac{\partial f}{\partial x_n} \end{bmatrix}
$$

## Geometric Interpretation

- $\nabla f$ points in the direction of **steepest ascent**
- $-\nabla f$ points in the direction of **steepest descent**
- $\|\nabla f\|$ gives the rate of change in that direction
- $\nabla f = 0$ at critical points (minima, maxima, saddle points)

The gradient is always perpendicular to the level curves (contour lines) of the function.

## Gradient Descent

The fundamental optimization algorithm:

$$
\mathbf{x}_{t+1} = \mathbf{x}_t - \alpha \nabla f(\mathbf{x}_t)
$$

where $\alpha$ is the **learning rate**.

### Learning Rate

- **Too large**: oscillation, divergence
- **Too small**: slow convergence
- **Just right**: smooth convergence to minimum

### Stochastic Gradient Descent (SGD)

Instead of computing the gradient over all data, use a random mini-batch:

$$
\mathbf{x}_{t+1} = \mathbf{x}_t - \alpha \nabla f_{\text{batch}}(\mathbf{x}_t)
$$

This introduces noise but dramatically reduces computation per step.

<Callout type="tip">Modern optimizers (Adam, RMSProp) build on gradient descent by adapting the learning rate per-parameter using running averages of past gradients.</Callout>

```python
import numpy as np

# Gradient descent on f(x) = x^2 + y^2
def f(x, y):
    return x**2 + y**2

def grad_f(x, y):
    return np.array([2*x, 2*y])

# Gradient descent
x, y = 5.0, 3.0
lr = 0.1
trajectory = [(x, y)]

for i in range(20):
    g = grad_f(x, y)
    x -= lr * g[0]
    y -= lr * g[1]
    trajectory.append((x, y))
    if np.linalg.norm(g) < 1e-6:
        break

print(f"Start: ({trajectory[0][0]:.2f}, {trajectory[0][1]:.2f})")
print(f"End: ({x:.6f}, {y:.6f})")
print(f"f at end: {f(x, y):.8f}")  # Should be ≈ 0
print(f"Steps: {len(trajectory)}")
```

## Key Takeaway

Gradients are the compass of machine learning optimization. They tell us which direction to update parameters to reduce loss. Understanding gradient descent and its variants (SGD, Adam, RMSProp) is fundamental to training any neural network.
